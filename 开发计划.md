# 瞬 Tempo — 完整开发计划

## 一、应用定位与命名

### 应用名称
**瞬 Tempo**（主名：**瞬**，英文：**Tempo**）

- **"瞬"** — 刹那、瞬间，捕捉时间的最小粒度
- **"Tempo"** — 节拍、节奏，表达时间的流动韵律
- 中英结合，简洁有力，符合 Apple 式命名哲学

### App 概念
一款 macOS 风格的高级日程 & 待办桌面应用。将日历的宏观视野与待办的微观执行统一在一个磨砂玻璃质感的界面中。

## 二、设计系统（Design System）

### 2.1 视觉风格
| 属性 | 方案 |
|------|------|
| 风格 | 苹果磨砂玻璃（Glassmorphism）+ 极简主义 |
| 主色调 | 系统蓝（\`#007AFF\`）为点缀色，大面积中性灰/白 |
| 玻璃效果 | \`backdrop-filter: blur(20px)\` + 半透明背景 |
| 圆角 | 统一 \`12px\`（大面板），\`8px\`（控件），\`6px\`（小元素） |
| 阴影 | 多层柔和阴影，Apple 式 \`shadow-sm/md/lg\` |
| 窗口控件 | 红 🔴 / 黄 🟡 / 绿 🟢 三圆点 |
| 字体栈 | \`-apple-system, BlinkMacSystemFont, "SF Pro Display", "Inter", "Helvetica Neue", sans-serif\` |

## 四、技术架构

### 4.1 技术栈选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 桌面框架 | **Electron 33** | 成熟，原生窗口，EXE 打包 |
| 打包器 | **electron-builder** | NSIS 安装器，产出 .exe |
| UI 框架 | **React 19 + TypeScript** | 组件化，生态丰富 |

| 动画 | **Framer Motion 12** | React 动画标杆 |
| 状态管理 | **Zustand** | 轻量、TS 友好 |
| 数据持久化 | **Dexie.js** (IndexedDB) | 零原生依赖，跨平台 |
| 图表 | **Recharts** | React 原生，动画支持好 |
| 图标 | **Lucide React** | 树摇优化，风格统一 |
| 日期处理 | **date-fns** | 轻量 Tree-shakable |
| 自然语言 | **chrono-node** | 支持中英文日期解析 |

## 六、关键决策

### Electron vs Tauri
| 对比项 | Electron | Tauri |
|--------|----------|-------|
| 成熟度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 社区资源 | 海量 | 较少 |
| EXE 体积 | ~150MB | ~5MB |
| 开发速度 | 快 | 中（需 Rust） |

选择 Electron：开发效率优先，体积在可接受范围。

### IndexedDB vs SQLite
SQLite 需原生编译模块，在 Electron 中配置复杂。IndexedDB + Dexie.js 零依赖，对 To-do 应用性能绰绰有余。

### 窗口控件
Electron `frame: false` + 自定义红/黄/绿三圆点，Windows 上模拟 macOS 风格。
